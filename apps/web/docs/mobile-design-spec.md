# Login / Register 移动端设计规范

## 1. 视口与安全区域
- 设计基线宽度：`375px`（iPhone 常见宽度）
- 页面最大安全宽度：`460px`
- 外边距：左右 `16px`，小屏（<=375）可降到 `12px`
- 最小触控区：`48px`

## 2. 字体与层级
- 字体族：`General Sans, Noto Sans SC, PingFang SC, sans-serif`
- 主标题：`clamp(32px, 9vw, 56px)`，字重 `700`
- 按钮文字：`30px`（小屏降为 `26px`），字重 `600`
- 正文：`14px - 16px`
- 标题字距：`-0.02em` ~ `-0.03em`

## 3. 颜色系统
- 页面背景：`#EDEDED`
- 主文本：`#0A0A0A`
- 主按钮：黑底白字（`#000000 / #FFFFFF`）
- 副按钮：透明底，黑色 1px 边框 + 黑字
- 错误反馈：浅红底 + 深红字
- 信息反馈：浅灰底 + 深灰字

## 4. 间距与圆角
- 间距刻度：`4 / 8 / 12 / 16 / 20 / 24 / 32`
- 卡片圆角：`22px`
- 控件圆角：`10px`
- 组件间默认垂直间距：`12px`

## 5. 组件规范
- 主按钮（Primary）
  - 高度：`>=48px`
  - 样式：黑底白字
- 副按钮（Secondary）
  - 高度：`>=48px`
  - 样式：透明底、黑色描边、黑字
- 输入框
  - 高度：`>=48px`
  - 支持 `focus-visible` 外轮廓

## 6. 交互与状态
- 所有可点击控件必须提供 `:focus-visible`
- `disabled` 状态降低透明度并禁用指针
- 错误/成功信息在按钮区下方统一展示
- 动效尽量轻量，按压反馈不超过 `120ms`

## 7. 响应式策略
- Mobile First
- `<=375px`：收紧内边距与按钮字体
- `>=640px`：在保留移动布局的前提下增加阴影和边框，避免无限拉伸
